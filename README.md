# EuroOpenCharts

FAIR, offline-first, HPC-ready research prototype for generating nautical chart and scientific data bundles for the **eastern Central-Southern Tyrrhenian Sea**.

The actual-data workflow produces outputs only from configured real source data or explicit omission metadata. Older prototype examples may use clearly marked fixtures, but production and high-quality runs never fabricate nautical data.

> Experimental scientific chart bundle. Not for primary navigation.

## What works now

- Unattended command-line workflow.
- Local DAG execution with parallel ready-task scheduling, mirroring DAGonStar task semantics.
- Offline generated bundle under `data/example`.
- Open formats for QGIS and Python: CSV, GeoJSON, JSON/STAC, SVG.
- Signal K resource export.
- OpenCPN supplemental GeoJSON prototype package.
- FAIR metadata and provenance files.
- Paper-ready SVG figure generated by code, not by GenAI.
- High-quality actual-data example with source freshness, resolution metadata, MPA rule validation, and OpenBridge SVG symbol configuration.

## Repository policy

`data/` is the root for generated and downloaded data and is intentionally ignored by Git.

## Actual-data-only run

The project now includes an actual-data-only pipeline:

```bash
python run_actual_data_only.py
```

This mode does **not** fabricate bathymetry, seamarks, ports, buoys, lights, labels, or anchorages. In the current offline execution environment it uses only real local geospatial datasets distributed with the installed Basemap data package: GSHHS coastlines and ETOPO1 relief imagery. Copernicus/EMODnet and OpenSeaMap layers must be supplied/downloaded as real datasets; the pipeline does not fall back to synthetic data.

## Highest-quality actual-data map example

The high-quality example is configured by `examples/highest_quality_map_config.json`:

```bash
python examples/create_highest_quality_map.py --config examples/highest_quality_map_config.json
```

The example supports config-driven downloads for authoritative/open bathymetry and seamark sources such as GEBCO, EMODnet, and OpenStreetMap/OpenSeaMap extracts. Replace the `source_downloads` URLs with official AOI-specific dataset URLs before making those downloads required. Missing optional downloads are recorded in the output metadata; missing required downloads fail fast. The renderer uses high-resolution GSHHS/ETOPO assets and produces PNG/PDF outputs with the required navigation warning.

For a production-quality run:

1. Download or export the latest official AOI subsets for GEBCO, EMODnet Bathymetry, OpenStreetMap/OpenSeaMap seamarks, and authoritative MPA geometry/rules.
2. Put those files under `data/sources/highest_quality/` or update the paths in `examples/highest_quality_map_config.json`.
3. Replace every `https://example.invalid/...` entry in `source_downloads` with the official URL used to acquire the source.
4. Set required high-value layers to `"required": true` only after their official source URL, license, provenance, CRS, and quality metadata are complete.
5. Keep the OpenStreetMap/OpenSeaMap harbor, marina, bay, anchorage, mooring, light, buoy, and beacon extract fresher than 30 days.
6. Keep MPA geometry and legal-rule files synchronized with the official authority; stale or incomplete MPA rules must fail or be explicitly omitted.
7. Export OpenBridge Icons as local SVG files under `resources/symbols/openbridge/`, following the license for the icon pack from [OpenBridge Icons](https://www.openbridge.no/cases/openbridge-icons).
8. Ensure the `symbols.dictionary` entries in the config point to those local SVG assets. Silent fallback symbols are forbidden.
9. Run:

```bash
python examples/create_highest_quality_map.py --config examples/highest_quality_map_config.json
```

The output bundle records source download manifests, high-resolution quality requirements, OpenBridge symbol configuration, MPA provenance, and the required warning: `Experimental scientific prototype. Not for navigation.`

## Single configuration file

All runtime behavior is controlled by `config.json`. Every script accepts `--config` or uses `EUROOPENCHARTS_CONFIG`. Operational paths, layer enablement, required/optional source policy, rendering switches, and backend settings must not be hardcoded.

## Marine Protected Areas

The project now includes a first-class MPA layer model. Authoritative MPA geometry and zone/rules files can be configured in `config.json`. The workflow validates required properties and fails rather than inventing MPA zones, permissions, or rules. See `docs/marine_protected_areas.md`.

## Governance

`AGENTS.md` is the normative engineering contract for contributors and agents. It formalizes actual-data-only behavior, FAIR enforcement, copyright-safe rendering, documentation quality requirements, and fail-fast source handling.

## Automated tests

The project includes unattended unit and contract tests formalized in `AGENTS.md` and documented in `docs/testing.md`.

```bash
python -m pytest -q
```

The tests enforce actual-data-only behavior, single-configuration usage, MPA validation, provenance/license checks, and the absence of synthetic placeholder layers in production bundles.
